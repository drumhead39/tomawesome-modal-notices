# Contributing to TomAwesome Modal Notices

Thank you for your interest in improving TomAwesome Modal Notices.

## Reporting Bugs

Before submitting a bug, please check the existing GitHub Issues to see whether it has already been reported.

When opening a new Issue, include:

- The plugin version
- WordPress and PHP versions
- The active WordPress theme
- Steps needed to reproduce the problem
- What you expected to happen
- What actually happened
- Relevant screenshots or error messages

Do not publicly report security vulnerabilities. Follow the instructions in [SECURITY.md](SECURITY.md) instead.

## Suggesting Features

Feature requests are welcome through GitHub Issues. Please explain:

- What you would like the plugin to do
- Why the feature would be useful
- How you imagine it working

## Contributing Code

1. Fork this repository.
2. Create a new branch for the change.
3. Make one focused change at a time.
4. Test the change with supported WordPress and PHP versions.
5. Submit a pull request explaining what changed and why.

Please do not include unrelated formatting or code changes in the same pull request.

## Coding Standards

Contributions should:

- Follow WordPress coding standards
- Sanitize and validate user input
- Escape output appropriately
- Use WordPress APIs whenever practical
- Preserve backward compatibility when possible
- Avoid external tracking, advertising, or telemetry
